三层匹配流水线

从粗到精，逐层筛选 — 无论候选人库有多大，总能精准定位最匹配的人才

Layer 1

规则过滤

硬性条件筛选，纯规则引擎，零延迟。学历、年龄、司龄、职级等范围过滤，Pandas 向量化执行。

11,000 → ~8,000 ×1

Layer 2

关键词+向量混合

中文分词加权打分 + Bailian text-embedding-v4 语义向量，双路并行融合，Top50 + 15 交叉补充。

~8,000 → 100 ×80

Layer 3

LLM 精排

qwen3.7-plus 阅读理解全部候选人摘要，逐条打分排序，输出评级 + 推荐理由。模拟资深 HR 招聘主管的判断。

100 → Top N ×10-20

FAQ

常见问题

关键词匹配和向量搜索的关系是什么？ +

两者并行执行、结果融合。关键词负责精确匹配（速度快、无延迟），向量负责语义扩展（理解同义词、近义词、相关概念）。关键词 Top50 + 向量 Top15 交叉补充，覆盖"精确命中"和"语义相似"两种场景。

首次搜索为什么比较慢？ +

首次使用需要预计算所有候选人的向量嵌入。400 条记录约需 6-8 秒（8 线程并行 + text-embedding-v4），结果缓存到磁盘。后续搜索从缓存加载，秒级响应。

支持哪些 LLM 后端？ +

三层后端可插拔：Bailian（阿里云百炼，qwen3.7-plus + text-embedding-v4，当前使用）、AIHub（公司内网 AI 平台，公网不可达）、Mock（开发阶段规则模拟，零 API 调用）。配置文件切换一行代码即可。

如何嵌入到现有的 HR 门户？ +

一行 script 标签即可：<script src="域名/widget/talent-widget.js" data-ws="wss://域名"></script>。Widget 使用 Shadow DOM 完全隔离样式，不会与宿主页面冲突。
